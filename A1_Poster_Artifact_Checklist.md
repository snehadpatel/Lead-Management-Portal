# A1 Poster Artifact Checklist

Use this with `A1_Studio_Project_Poster_Template.docx`.

## Mandatory Rules
- Poster size: **A1**
- Content style: **concise bullet points only**
- Visual balance: **60% visuals + 20% text + 20% white space**
- Main focus: **Tri-Course Integration Architecture**

## 1) Header Section
- Project title (clear and technical)
- Team members + enrollment numbers
- Guide/mentor name
- Program/semester/institute
- Optional QR to demo/repo

## 2) Introduction & Problem Statement
- Domain context in 1-2 bullets
- Exact problem statement (what fails today)
- Why this problem matters to industry
- 3 clear objectives (action verbs only)

## 3) Tri-Course Integration Architecture (Largest Panel)
- Big Data layer: ingestion, storage, preprocessing
- AI layer: model training, inference, analytics
- Deployment/Cloud layer: API, UI, runtime, containers
- End-to-end flow arrows: source -> ETL -> model -> serving -> output
- One color legend for all three layers

## 4) Methodology & Prototype Implementation
- Tech stack bullets (only tools actually used)
- Pipeline steps: ingest -> transform -> train -> serve
- Prototype components: backend API + dashboard + data pipeline
- Reproducibility note: metric files / evaluation reports

## 5) Results & Validation
- Quantitative metrics table (accuracy/F1/AUC, silhouette, etc.)
- 3-4 visuals: confusion matrix, cluster chart, dashboard screenshot, forecast plot
- Validation bullets: train/test or CV approach
- Mention exact metric source files

## 6) Conclusion & Future Scope
- 3 key outcomes (integration-focused)
- 1-2 practical limitations
- 3 future enhancements (scale, real-time, better models, MLOps)

## Visual Placement Plan (Quick)
- Center (largest): Tri-course architecture
- Side/top: intro + methodology
- Right/bottom: results visuals + KPI mini table
- Footer: conclusion + future scope

## Final QA Before Print
- Are all 6 required sections present?
- Is architecture visually dominant?
- Are there measurable validation numbers?
- Is text short enough to read from distance?
- Is whitespace preserved (not overcrowded)?
